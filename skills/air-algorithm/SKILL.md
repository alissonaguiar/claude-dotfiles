# AI/R Algorithm — Agentic Enterprise Reinvention Framework

## Overview

Apply AI/R's 5-step reinvention framework to any idea, app, software, workflow, or initiative. The algorithm moves in deliberate sequence: **clarity → subtraction → redesign → scale**. Never skip steps. Never automate before simplifying.

> Before you scale, simplify. Before you automate, redesign. Before you transform, define the outcome.

---

## How to Apply This Skill

When the user submits something for analysis, run all 5 steps in order. Present each step as a section with clear findings and recommendations. Be direct, specific, and actionable — not generic.

---

## Step 1 — Define the Goal in One Simple Phrase

**Objective:** Extract or propose one single clear business/product outcome.

- Ask: *What is the ONE thing this needs to achieve?*
- Reject broad ambitions, lists of goals, or vague vision statements.
- If the user didn't specify, propose 2–3 candidate goal phrases and ask them to choose one.

**Output format:**
```
GOAL: [one sentence — verb + measurable outcome]
Example: "Reduce onboarding time from 2 weeks to 3 days."
```

---

## Step 2 — Challenge Every Requirement Against the Goal

**Objective:** Question every feature, step, screen, rule, approval, or process that exists in the submitted idea/system.

For each identified requirement, challenge it:
- Why does this exist?
- What value does it create toward the goal?
- Who does it serve?
- Does it move toward or away from the goal?

**Output format:** A table or list:
```
| Requirement          | Justification      | Verdict         |
|---------------------|--------------------|-----------------|
| User login screen   | Security + identity| KEEP            |
| Weekly email report | Legacy habit       | CHALLENGE       |
| PDF export feature  | Rarely used        | CHALLENGE       |
```

---

## Step 3 — Delete What Does Not Need to Exist

**Objective:** Remove anything that failed Step 2. Be decisive.

- Eliminate features, steps, screens, approvals, or workflows with no clear goal alignment.
- Mark what should be deleted, deferred, or merged.
- Warn if the user is trying to optimize something that should be removed entirely.

**Output format:**
```
DELETE:  [list of items to remove and why]
DEFER:   [items not needed now but potentially later]
MERGE:   [items that can be combined]
```

---

## Step 4 — Redesign Around People + Agents

**Objective:** Redesign what remains for a world where intelligent agents are available.

Ask the core reinvention question:
> *If we were designing this today, with Agentic AI available, how should it work?*

Define the new model:
- **People do:** judgment, accountability, intent, exceptions, context
- **Agents do:** analysis, orchestration, coordination, execution, monitoring

Identify where agents should take responsibility and where humans stay in the loop. Redesign roles, decision points, and workflow accordingly.

**Output format:**
```
HUMAN RESPONSIBILITIES:
- [what people own]

AGENT RESPONSIBILITIES:
- [what agents handle]

REDESIGNED WORKFLOW:
- [step-by-step flow with human/agent ownership marked]
```

---

## Step 5 — Use Agentic AI to Accelerate, Automate, Scale, and Continuously Improve

**Objective:** Identify where Agentic AI creates the most leverage on the redesigned workflow.

Focus on:
- Compressing cycle time
- Reducing manual coordination
- Automating repeatable execution
- Expanding capacity without adding headcount
- Creating feedback loops for continuous improvement

Suggest specific AI/agent capabilities that apply (e.g., context preservation, multi-step reasoning, system integration, monitoring, exception handling).

**Output format:**
```
ACCELERATE: [where agents speed things up]
AUTOMATE:   [what agents fully own]
SCALE:      [how the model grows without proportional effort]
IMPROVE:    [feedback loops and continuous refinement]
```

---

## Final Output Structure

After all 5 steps, close with:

```
## AI/R Algorithm Summary — [Subject Name]

GOAL: [one phrase]

SIMPLIFIED: [what was removed/deferred]

REDESIGNED MODEL: [people + agents — 3–5 bullet points]

AGENTIC LEVERAGE: [top 3 AI applications]

NEXT ACTION: [most important first step to take]
```

---

## Common Mistakes

| Mistake | Correction |
|--------|------------|
| Jumping to automation before simplifying | Always run Step 3 (delete) before Step 5 (automate) |
| Keeping vague goals | Force one phrase — no lists, no ambiguity |
| Treating all features as sacred | Everything must justify itself against the goal |
| Making agents do everything | People remain central — define clear human ownership |
| One-time redesign thinking | Step 5 must include feedback loops for continuous improvement |

---

## Tone & Style

- Be analytical and direct, not diplomatic about what should be cut.
- Use tables and lists — avoid long paragraphs in output.
- Always close with a concrete **Next Action** — the single most important move.
